## 1. What is Flutter

Flutter is Google's open-source UI framework for building apps from a single codebase that runs on Android, iOS, Web, Windows, macOS, and Linux.

It uses the **Dart** programming language. Instead of using native UI components, Flutter draws its own widgets on a canvas, which makes it fast and consistent across platforms.

In this project, Flutter is the frontend — it shows the form, handles user input, calls the backend, and displays results.

---

## 2. What is FastAPI

FastAPI is a modern Python web framework for building APIs quickly. It is:

- Fast to write (minimal boilerplate)
- Automatically generates API documentation at `/docs`
- Uses Python type hints for request/response validation via **Pydantic**
- Supports async/await for non-blocking operations

In this project, FastAPI acts as the backend server. Flutter sends feedback data to it, the ML model runs inside it, and it returns the predicted category back to Flutter. It also triggers the thank-you email.

To run it:
```bash
uvicorn main:app --reload
```
This starts the server at `http://127.0.0.1:8000`. The `--reload` flag auto-restarts when you change code.

### Running FastAPI for phone access

When testing on a physical phone, use:
```bash
uvicorn main:app --host 0.0.0.0 --port 8000 --reload
```

`--host 0.0.0.0` tells the server to listen on all network interfaces — not just the local machine. This allows your phone on the same network to reach the server. `127.0.0.1` only accepts connections from the same machine. `0.0.0.0` accepts from any device on the network.

### Windows Firewall

By default Windows blocks incoming connections on port 8000. To allow it, run PowerShell as Administrator:
```powershell
New-NetFirewallRule -DisplayName "FastAPI 8000" -Direction Inbound -Protocol TCP -LocalPort 8000 -Action Allow
```

## 3. What is Firebase & Firestore

**Firebase** is Google's Backend-as-a-Service (BaaS) platform. It provides ready-made backend features like databases, authentication, storage, and hosting — without writing server code.

**Cloud Firestore** is Firebase's NoSQL database. Data is stored as **collections** (like folders) containing **documents** (like JSON objects).

In this project:
- Firebase is initialized in `main.dart` using `firebase_options.dart` (generated by FlutterFire CLI)
- After the ML model predicts a category, Flutter saves the feedback to the matching Firestore collection
- Three collections are used: `complaints`, `compliments`, `feature_requests`

Example document stored in Firestore:
```json
{
  "name": "Yash",
  "email": "yash@gmail.com",
  "message": "The app keeps crashing",
  "category": "Complaint",
  "timestamp": "2026-05-22T11:00:00Z"
}
```

## 4. What is the ML Model

The ML model is a text classification model. It reads a feedback message and predicts which of three categories it belongs to.

**Algorithm used: TF-IDF + Logistic Regression**

- **TF-IDF (Term Frequency-Inverse Document Frequency)**: Converts text into numbers. Words that appear often in one category but rarely in others get higher scores. This helps the model understand which words are important.

- **Logistic Regression**: A classification algorithm that learns from labeled examples. Given the TF-IDF numbers, it learns patterns like "words such as crash, broken, hate → Complaint".

- **SentimentFeatures (custom)**: Because the dataset had noisy mixed sentences, a custom feature extractor was added. It counts how many complaint/compliment/feature-request keywords appear in the text and adds those counts as extra signals to the model.

- **FeatureUnion**: Combines TF-IDF output and SentimentFeatures output into one combined feature vector before passing to Logistic Regression.

The model is trained once and saved as `model.pkl`. On every server restart it loads from pkl instead of retraining.

## 5. What is the Dataset

File: `backend/large_feedback_dataset.csv`

- 9,000 rows total
- 3,000 rows per category (balanced)
- Two columns: `feedback` (text) and `category` (label)

Example rows:
```
feedback,category
"The app keeps crashing every time I open it",Complaint
"Fantastic user experience, very smooth",Compliment
"Please add dark mode feature",Feature Request
```

The dataset had some noise — compliment rows contained complaint-like phrases like "please fix". This caused the original model to misclassify. The fix was adding the `SentimentFeatures` class which uses keyword matching to override the noisy patterns.

## 6. Flutter — lib/ folder explained

### main.dart

The entry point of the Flutter app.

```dart
void main() async {
  WidgetsFlutterBinding.ensureInitialized();   // Required before any async work
  await Firebase.initializeApp(...);            // Connect to Firebase
  runApp(const MyApp());
}
```

`ChangeNotifierProvider` wraps the entire app so `FeedbackProvider` is accessible from any screen. `SplashScreen` is set as the first screen.

---

### screens/splash_screen.dart

The opening screen shown for 2 seconds when the app launches.

- Uses `initState()` to start a `Future.delayed` timer
- After 2 seconds, navigates to `FeedbackScreen` using `Navigator.pushReplacement` (so the user can't go back to splash)
- Shows the app name and a feedback icon

---

### screens/feedback_screen.dart

The main form screen where users enter their feedback.

- Has three `TextEditingController`s for name, email, and message
- Uses a `Form` widget with a `GlobalKey` for validation
- On submit, calls `FeedbackProvider.submitFeedback()`
- Watches provider status — if `success`, navigates to `SuccessScreen`; if `error`, shows a red snackbar
- The submit button shows a loading spinner while the request is in progress

---

### screens/success_screen.dart

Shown after feedback is successfully submitted.

- Receives the predicted `category` as a constructor parameter
- Shows a different icon based on category:
  - Complaint → warning icon
  - Compliment → thumbs up icon
  - Feature Request → lightbulb icon
- Shows a blue badge with the category name
- "Submit Another" button resets the provider and goes back to the form

---

### providers/feedback_provider.dart

The brain of the Flutter app. Manages state and orchestrates the full submission flow.

```
submitFeedback() called
  → status = loading  (UI shows spinner)
  → ApiService.predictCategory()  (calls FastAPI)
  → FirebaseService.saveFeedback()  (saves to Firestore)
  → status = success  (UI navigates to success screen)
```

Uses `ChangeNotifier` — when `notifyListeners()` is called, all widgets watching this provider rebuild automatically.

`FeedbackStatus` enum has four states: `idle`, `loading`, `success`, `error`.

---

### services/api_service.dart

Handles HTTP communication with the FastAPI backend.

```dart
final response = await http.post(
  Uri.parse('http://127.0.0.1:8000/predict'),
  headers: {'Content-Type': 'application/json'},
  body: jsonEncode({'name': ..., 'email': ..., 'message': ...}),
);
```

Parses the JSON response and returns the `category` string. Throws an exception if the status code is not 200.

---

### services/firebase_service.dart

Handles all Firestore database operations.

```dart
await _firestore.collection(collection).add({
  'name': feedback.name,
  'email': feedback.email,
  'message': feedback.message,
  'category': feedback.category,
  'timestamp': FieldValue.serverTimestamp(),
});
```

`_collectionFor()` maps the category string to the correct Firestore collection name using a switch statement. `FieldValue.serverTimestamp()` uses Firebase's server time instead of the device clock.

---

### models/feedback_model.dart

A plain Dart class that represents one feedback entry.

- Holds: `name`, `email`, `message`, `category`, `timestamp`
- `toJson()` converts it to a Map for Firestore
- `category` and `timestamp` are nullable because they are set after the ML prediction

---

### widgets/custom_textfield.dart

A reusable text input widget that wraps Flutter's `TextField`.

Accepts: `label`, `controller`, `maxLines` (default 1), `keyboardType` (default text).

Used in `FeedbackScreen` for all three input fields. Keeps the form code clean by avoiding repetition.

---

### widgets/custom_button.dart

A reusable full-width button with a built-in loading state.

When `isLoading` is true, it shows a `CircularProgressIndicator` instead of the label text and disables the `onPressed` callback to prevent double-taps.

---

### utils/constants.dart

A single place for all hardcoded values used across the app.

```dart
static const String apiBaseUrl = 'http://127.0.0.1:8000';
static const String complaintsCollection = 'complaints';
static const String complimentsCollection = 'compliments';
static const String featureRequestsCollection = 'feature_requests';
```

If you deploy the backend to a server, you only need to change `apiBaseUrl` here.

## 7. Backend — backend/ folder explained

### main.py

The FastAPI application file. Defines the API server and its endpoints.

```python
@app.post("/predict", response_model=PredictionResponse)
async def predict(feedback: FeedbackRequest):
    category = predict_category(feedback.message)   # ML prediction
    send_thank_you_email(...)                        # Email
    return PredictionResponse(category=category)    # Response to Flutter
```

`CORSMiddleware` is added so the Flutter web app (running on a different port) is allowed to call this API. Without CORS, the browser would block the request.

`FeedbackRequest` and `PredictionResponse` are Pydantic models — they automatically validate incoming JSON and serialize outgoing responses.

---

### ml_model.py

Contains all ML logic: training, saving, loading, and predicting.

**Key functions:**

`_clean(text)` — lowercases text, removes punctuation, strips extra spaces.

`SentimentFeatures` — a custom scikit-learn transformer. For each text, it counts how many complaint/compliment/feature keywords appear and returns those counts as a numeric array. This is combined with TF-IDF to improve accuracy on noisy data.

`_train_and_save()` — loads the CSV, cleans text, splits into train/test (80/20), builds the pipeline, trains it, prints accuracy report, saves to `model.pkl`.

`_load_model()` — checks if `model.pkl` exists. If yes, loads it. If no, calls `_train_and_save()`. Uses a module-level `_pipeline` variable so the model is only loaded once per server session.

`predict_category(message)` — the public function called by `main.py`. Cleans the input and returns the predicted label.

---

### email_service.py

Sends an HTML thank-you email to the user via Gmail SMTP.

**How Gmail SMTP works:**
1. Connect to `smtp.gmail.com` on port 587
2. Start TLS encryption (`starttls()`)
3. Login with Gmail address and App Password
4. Send the email

The email is HTML formatted — it shows the user's name, a thank-you message, and the feedback category in a styled card.

Credentials are loaded from `backend/.env` using `python-dotenv`. If credentials are not set, the function prints a warning and skips sending (so the API still works during development).

---

### large_feedback_dataset.csv

The training data for the ML model. 9,000 rows, two columns: `feedback` and `category`.

The model is trained on this file the first time the server starts. After training, `model.pkl` is saved and the CSV is no longer needed at runtime.

---

### model.pkl

The serialized (saved) trained ML pipeline. Created automatically by `ml_model.py` after first training.

Contains the entire pipeline: TF-IDF vectorizer + SentimentFeatures transformer + Logistic Regression classifier — all saved together so predictions work without retraining.

To force a retrain, delete this file and restart the server.

---

### requirements.txt

Python package dependencies for the backend.

```
fastapi       — web framework
uvicorn       — ASGI server to run FastAPI
pydantic      — data validation
python-dotenv — loads .env file
scikit-learn  — ML algorithms (TF-IDF, Logistic Regression)
pandas        — reads the CSV dataset
```

---

### .env

Stores sensitive credentials. Never committed to Git.

```
SMTP_USER=your_email@gmail.com
SMTP_PASS=your_app_password
```

Loaded by `email_service.py` using `load_dotenv()`.

## 8. How Everything Connects

```
[Flutter App]
     |
     | User fills form and taps Submit
     |
     ↓
[FeedbackProvider]
     |
     | Calls ApiService.predictCategory()
     |
     ↓
[FastAPI /predict endpoint]  ←── backend/main.py
     |
     | Calls predict_category(message)
     |
     ↓
[ML Model]  ←── backend/ml_model.py
     |
     | Returns: "Complaint" / "Compliment" / "Feature Request"
     |
     ↓
[FastAPI]
     |
     | Calls send_thank_you_email()
     |
     ↓
[Gmail SMTP]  ←── backend/email_service.py
     |
     | Sends HTML email to user
     |
     ↓
[FastAPI returns category to Flutter]
     |
     ↓
[FeedbackProvider]
     |
     | Calls FirebaseService.saveFeedback()
     |
     ↓
[Firebase Firestore]  ←── lib/services/firebase_service.dart
     |
     | Saves document to complaints / compliments / feature_requests
     |
     ↓
[FeedbackProvider sets status = success]
     |
     ↓
[FeedbackScreen navigates to SuccessScreen]
     |
     ↓
[SuccessScreen shows category + icon + thank you message]
```
